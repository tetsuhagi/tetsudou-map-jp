# 記事執筆依頼プロンプトテンプレ

新規記事を依頼するときに使う、コピペ用の **標準プロンプト** 。
スレッドが長くなってコンテキストが圧縮されても、必要な情報を毎回 fresh に取得できるよう
docs 再読を強制し、重要ルールを inline で再掲する設計。

---

## 標準版（コピペ用）

```text
【鉄道マップ全国版 記事執筆依頼】

# 入力（空欄は適切に判断してOK）
記事タイプ:           ※ ニュース / 路線詳細 / 旅行記 / 雑記
タイトル案:           ※ 空欄OK。入力されてもSEO/SNS拡散の観点でより良い案がある場合は改変する
内容:                 ※ 「これこれこういうことを解説してほしい」を自由記述。タイトル案より優先される一次情報源として扱う
slug:
カテゴリ:             ※ 寝台特急 / 新幹線 / 特急 / 私鉄特急 / 鉄道ニュース / 旅行・観光ニュース など
公開日:               ※ ニュースは必須（未指定なら今日の日付）
運行区間:             ※ 路線詳細時のみ
サムネイル画像:        ※ 【現行方針: 非表示】 サイト全体でサムネ表示はオフ中。指定されても記事には埋め込まない（workflow.md §1.6 参照）
備考:                  ※ 兄弟記事・記事の角度・特記事項など

# 参照URL（ニュース系は必須・他カテゴリでも参考資料があれば）
主情報源（PR/公式発表）:
  -
追加情報源（中立報道・関連解説）:
  -
  -

# 着手前プロセス（スレッドが長くても毎回守ること）
1. 以下5ファイルを必ず再読:
   - docs/voice-guide.md
   - docs/article-workflow.md
   - docs/article-template.html      ※ 路線詳細・旅行記・雑記
   - docs/article-template-news.html ※ ニュース
   - docs/articles-index.md
2. 参照URLがあれば WebFetch で内容を取得し、要約・整理してから執筆
3. 既存の本番記事を語感サンプルとして確認:
   routes/sunrise-seto.html / routes/sunrise-izumo.html

# 違反NGの重要ルール
- バルーンは自動生成して埋めてOK（2026-05-22〜）。生成後 docs/balloon-review-queue.md に記録。依頼者がバルーン文言を指定した場合はそれを最優先
- 具体時刻（XX:XX）は書かない。頻度＋公式時刻表リンクで誘導
- data/timetables の CSV は参考扱い、記事に転記しない
- 参照URLの文章はコピペ禁止、要約・解釈のみ。出典は本文中に明示し、本文末に元リンクを置く
- ニュースは公開日を冒頭 page-meta と disclaimer に明記
- 既存記事との相互リンクは docs/articles-index.md を見て両方向に張る

# 完了時の更新セット
- 路線詳細: routes/{slug}.html / routes/index.html / sitemap.xml / _redirects / docs/articles-index.md / index.html (TOPの最新カード)
- コラム・雑記・旅行記: columns/{slug}.html / columns/index.html / sitemap.xml / _redirects / docs/articles-index.md / index.html (TOPの最新カード)
- ニュース: news/{slug}.html / news/index.html / sitemap.xml / _redirects / docs/articles-index.md / index.html (TOPの最新カード)
- サムネ: assets/og/{slug}.jpg（指定があれば）+ HTML の og:image系設定 + 記事冒頭の article-hero + TOPの article-card 画像
```

## 入力欄が空欄だったときの判断ルール

| 欄 | 空欄時の判断 |
|---|---|
| 記事タイプ | 内容・タイトル・URL・備考から推測（ニュース性ならニュース、列車名主体なら路線詳細、考察・意見・体験ならコラム） |
| タイトル案 | **内容欄を主たる素材** にして、SEO（検索クエリとの一致）と SNS 拡散（フック・興味喚起）を意識して生成。「{主題} | {サブタイトル：完全ガイド / 〇〇を考察 / 〜のカギ？ など}」型を基本にしつつ、内容に応じて柔軟にアレンジ |
| タイトル案（入力あり） | 入力されたタイトルも、より良いSEO/SNS最適化版が作れる場合は改変する（依頼者は「タイトル考えるのが苦手」と明言済み）。改変した場合は冒頭で「タイトル案を○○に変更しました」と一言断る |
| 内容 | タイトル案・備考・参照URLから記事の方向性を推測。すべて空欄なら「記事の方向性が不明」と確認を求める |
| slug | タイトル（または内容の主題）から kebab-case で生成。articles-index.md と重複しないこと |
| カテゴリ | 記事タイプから自動マッピング（路線詳細 → 寝台特急/新幹線/etc、コラム → コラム・考察/旅行記、ニュース → 鉄道/旅行・観光） |
| 公開日 | 今日の日付（YYYY-MM-DD） |
| 運行区間 | 路線詳細時に空欄なら、参照URL or 一般知識から記載 |
| サムネイル画像 | **現行方針: 非表示**。指定の有無に関わらず og:image / twitter:image / JSON-LD image / article-hero / TOP card の image div はすべて含めない。`assets/og/{slug}.jpg` への圧縮配置自体は希望があれば実施可（将来再開時に使える） |
| 備考 | 特記なしとして進める |
| 参照URL | ニュース系で空欄 → 「参照URLが必要」と確認を求める／路線詳細・コラムでは空欄でOK |

**最低限必要な情報**: 「タイトル案」または「内容」のどちらか一方は必ず必要。両方空欄なら、執筆前に確認を求める。

**タイトル生成時のSEO/SNS最適化ポイント**:
- 検索ボリュームのあるキーワードを含める（列車名・地名・「完全ガイド」「アクセス」「ランキング」「比較」等）
- 数字・疑問形・比較表現を活用してクリック率を上げる（例: 「3つの理由」「〜のカギ？」「〜vs〜」）
- 冗長な接続語（「について」「に関して」等）は削る
- 30〜40字程度を目安にしつつ、サブタイトル付きで情報量を確保
- 既存の本番記事タイトル（例: 「サンライズ瀬戸 完全ガイド｜停車駅・時刻・個室・予約のコツ」「リニアは博多まで来るのか？｜延伸の可能性と現実の壁を考察」）の型を参考にする

## サムネイル画像の処理（2026年5月 現在: 非表示方針）

> **現行方針**: AI生成サムネが「AI製感」でユーザーを遠ざける懸念があるため、サイト全体でサムネ表示はオフ。
> 詳細は `docs/article-workflow.md §1.6` を参照。

サムネ指定があっても、HTML には **og:image / twitter:image / JSON-LD image / article-hero / TOP card image を含めない**。
画像ファイル自体を `assets/og/{slug}.jpg` に置きたい場合は実施可（将来再開時に使える）。
